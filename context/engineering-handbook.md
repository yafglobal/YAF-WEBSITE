Our Engineering Values

These aren't aspirational posters. These are the principles we actually use to make decisions:

1. Write code for the next person.

You're not the only one who'll read it. Write clear variable names, add comments for non-obvious logic, and structure your code so someone unfamiliar can follow it.

2. Ask for help early.

Being stuck for 30 minutes and asking is strength. Being stuck for 4 hours in silence is a problem. Post in #engineering or message someone directly.

3. Ship small, ship often.

Small PRs get reviewed faster, break less, and are easier to reason about. If your PR is over 400 lines, you should probably split it.

4. Leave the code better than you found it.

If you touch a file and notice something messy, clean it up. Add a missing type. Fix a misleading variable name. Small improvements compound.

5. Default to transparency.

If you're blocked, say so. If you made a mistake, own it. If you disagree with a decision, speak up respectfully. No one gets in trouble for honesty.
